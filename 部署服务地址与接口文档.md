# 小乐语音助手部署服务地址与接口文档

## 1. 服务地址

当前已部署并运行的前端与接口服务：

| 项目 | 地址 | 说明 |
| --- | --- | --- |
| 前端访问地址 | `http://127.0.0.1:7860/` | Gradio 前端页面，用于语音/文字输入、数字人展示、聊天窗口与信息预览 |
| 健康检查接口 | `http://127.0.0.1:7860/mock/health` | 检查服务是否正常启动，并返回 mock 接口字段说明 |
| 语音聊天接口 | `http://127.0.0.1:7860/mock/voice-chat` | 接收用户文本、音频路径、角色、模式等参数，返回 ASR 文本、助手回复、TTS 地址和信息预览 |
| 数字人预览页面 | `http://127.0.0.1:7860/live2d-viewer?model=hiyori_pro_zh` | 数字人模型预览页面，支持角色模型切换 |

如果通过内网穿透或反向代理访问，将 `http://127.0.0.1:7860` 替换为公网域名即可，例如：

```text
https://你的域名/
https://你的域名/mock/health
https://你的域名/mock/voice-chat
```

## 3. 接口说明

### 3.1 健康检查接口

**请求地址**

```http
GET /mock/health
```

**完整地址**

```text
http://127.0.0.1:7860/mock/health
```

**请求参数**

无。

**返回字段**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | boolean | 服务是否正常 |
| `service` | string | 服务名称 |
| `endpoint` | string | 主要语音聊天接口路径 |
| `fields` | array | 语音聊天接口返回字段列表 |

**返回示例**

```json
{
  "success": true,
  "service": "xiaole-gradio-mock",
  "endpoint": "/mock/voice-chat",
  "fields": [
    "success",
    "request_id",
    "asr_text",
    "reply_text",
    "tts_audio_url",
    "crm",
    "tool_status",
    "timings_ms",
    "trace"
  ]
}
```

### 3.2 语音聊天接口

**请求地址**

```http
POST /mock/voice-chat
```

**完整地址**

```text
http://127.0.0.1:7860/mock/voice-chat
```

**请求头**

```http
Content-Type: application/json; charset=utf-8
```

**请求参数**

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `text` | string | 否 | 用户输入的文字内容；如果为空，则使用音频路径模拟 ASR 结果 |
| `audio_path` | string/null | 否 | Gradio 上传或录制后的本地音频路径 |
| `audio_summary` | array/string | 否 | 音频摘要信息，例如时长、文件大小 |
| `mode` | string | 否 | 对话模式，当前固定为 `日常聊天` |
| `voice_style` | string | 否 | 音色名称，由前端按角色自动绑定 |
| `char_name` | string | 否 | 角色名称，当前支持 `小乐（默认助手）`、`名取同学`、`胡桃小朋友` |
| `client` | string | 否 | 客户端标识 |
| `timestamp` | string | 否 | 请求时间 |

**请求示例**

```json
{
  "text": "胡桃你现在能说话吗",
  "audio_path": null,
  "mode": "日常聊天",
  "char_name": "胡桃小朋友",
  "client": "gradio-stage-1",
  "timestamp": "2026-06-21 16:00:00"
}
```

**返回字段**

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | boolean | 本次请求是否成功 |
| `request_id` | string | 请求编号 |
| `asr_text` | string | ASR 识别文本；当前 mock 模式下优先返回 `text` |
| `reply_text` | string | 助手回复文本 |
| `tts_audio_url` | string | TTS 音频地址；当前为 mock 占位地址 |
| `crm` | object | 信息预览字段 |
| `tool_status` | object | ASR、LLM、TTS、CRM 模块状态 |
| `timings_ms` | object | 模拟耗时 |
| `trace` | array | 请求处理步骤 |

**返回示例**


## 4. 调用示例

### 4.1 PowerShell 调用健康检查

```powershell
Invoke-RestMethod -Uri http://127.0.0.1:7860/mock/health
```

### 4.2 PowerShell 调用语音聊天接口

```powershell
$body = @{
  text = "你好小乐，帮我测试接口"
  mode = "日常聊天"
  char_name = "小乐（默认助手）"
  client = "test-client"
  timestamp = "2026-06-21 16:00:00"
} | ConvertTo-Json -Compress

Invoke-RestMethod `
  -Uri http://127.0.0.1:7860/mock/voice-chat `
  -Method Post `
  -Body ([System.Text.Encoding]::UTF8.GetBytes($body)) `
  -ContentType "application/json; charset=utf-8"
```

### 4.3 curl 调用语音聊天接口

```bash
curl -X POST "http://127.0.0.1:7860/mock/voice-chat" \
  -H "Content-Type: application/json; charset=utf-8" \
  -d '{
    "text": "你好小乐，帮我测试接口",
    "mode": "日常聊天",
    "char_name": "小乐（默认助手）",
    "client": "test-client",
    "timestamp": "2026-06-21 16:00:00"
  }'
```

## 6. 后续真实后端接入说明

当前 `/mock/voice-chat` 为前端联调接口，返回 mock 数据。后续接入真实小乐语音核心后端时，可将该接口内部逻辑替换为：

1. 接收前端音频或文字。
2. 调用小乐语音核心后端 ASR 模块获取识别文本。
3. 调用 LLM 生成回复。
4. 调用 TTS 生成语音。
5. 返回 `asr_text`、`reply_text`、`tts_audio_url`、`tool_status` 等字段，保持前端字段结构不变。

小乐语音核心后端基于 ESP32 语音链路提取，原始兼容服务默认配置地址如下，启动后可作为后续接入目标：

| 服务 | 默认地址 |
| --- | --- |
| WebSocket 服务 | `ws://127.0.0.1:8000/xiaozhi/v1/` |
| OTA 接口 | `http://127.0.0.1:8003/xiaozhi/ota/` |
| 视觉分析接口 | `http://127.0.0.1:8003/mcp/vision/explain` |
